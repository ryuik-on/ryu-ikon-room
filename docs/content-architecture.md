# Content architecture

この文書はセクションの役割だけを定義します。レイアウト、色、書体、アニメーション、実装技術は確定しません。

## 1. Hero

- 目的: サイト訪問直後に「ROOMという音楽公演」と日程を認識させる
- ユーザーが得る情報: 公演名、音楽公演であること、2026年11月22日
- 次のセクションへの役割: 写真・映像による公演体験へ自然に導く
- 必要素材: ROOMロゴまたは正式な表記、Hero用静止画または動画、代替表示
- 未確定事項: 静止画/動画、コピー、ロゴ、クレジット、動画自動再生の可否

## 2. 写真または映像

- 目的: 説明より先に奏者・観客・会場の熱と緊張感を伝える
- ユーザーが得る情報: ROOMで生まれる人と音の空間の具体像
- 次のセクションへの役割: 感覚的な体験からコンセプトの言葉へ接続する
- 必要素材: 採用写真または映像、poster画像、alt/代替説明、権利・クレジット情報
- 未確定事項: 採用点数、順序、映像尺、音声、自動再生、モバイル用素材

## 3. ROOMのコンセプト／Founder’s Note

- 目的: ROOMを開催する理由と、奏者から観客へ届けたいものを言葉で示す
- ユーザーが得る情報: 指定コンセプト原文、必要に応じて確定済みFounder’s Note
- 次のセクションへの役割: 感情的な納得を、参加判断に必要な公演情報へつなぐ
- 必要素材: 承認済み原稿、改行・表記ルール、必要なら署名または肩書
- 未確定事項: Founder’s Note最終稿、コンセプトとの併載/役割分担、署名

## 4. 公演情報

- 目的: 参加判断に必要な確定事実を短時間で確認できるようにする
- ユーザーが得る情報: 公演名、主催、日付、OPEN、START、会場、料金
- 次のセクションへの役割: 内容を理解したユーザーをチケット行動へ導く
- 必要素材: 確定情報、会場の正式表記、必要なら地図導線
- 未確定事項: 終演時刻、当日券、小人料金、出演者、セットリスト

## 5. チケット導線

- 目的: 迷わず正式な販売先へ到達させる
- ユーザーが得る情報: 料金、販売状態、購入方法、外部サイトへ移動すること
- 次のセクションへの役割: 購入しない/できないユーザーにも最新情報の入手先を示す
- 必要素材: 有効性を確認した正式URL、販売期間、注意事項、ボタン文言
- 未確定事項: 現行teket URLの代替、販売開始日、当日券、売切時表示

## 6. Instagram・公式サイトへの導線

- 目的: 最新情報、主催者情報、問い合わせ先へつなぐ
- ユーザーが得る情報: 公式Instagramと公式サイトの所在
- 次のセクションへの役割: ページを離れる前に継続接点を提供する
- 必要素材: 確認済みURL、リンク名、必要なら外部リンク表示
- 未確定事項: Ryuik-on/部の公式サイトURL、問い合わせ導線、SNS追加有無

## 7. Footer

- 目的: 主催者・権利表記と最終導線を簡潔にまとめる
- ユーザーが得る情報: 主催者名、著作権表記、必要なポリシー/連絡先
- 次のセクションへの役割: ページの終端を明確にし、再度主要導線へ戻れるようにする
- 必要素材: 正式な主催者表記、著作権年、必要な法的情報
- 未確定事項: 問い合わせ先、プライバシー表記、トップへ戻る導線

## セクション間の原則

- 体験（Hero・写真/映像）→意味（コンセプト）→事実（公演情報）→行動（Ticket）→継続接点（SNS/公式サイト）の順を維持する
- 同じ情報の反復は、5秒認知やチケット到達に必要な範囲へ限定する
- v1の細かなスクロール構成や演出順は引き継がない
