# Acceptance criteria — ROOM v2

この文書は公開可能と判断するための基準案です。技術選定前のため、コマンド名や実装方式は固定しません。

## 必須基準

### 内容と導線

- モバイルファーストで設計されている
- 最初の5秒で「ROOM」と「2026年11月22日」を認識できる
- 初見の人にも音楽公演であることが伝わる
- 主要画面幅でチケット導線まで迷わず到達できる
- 未確認情報を公開していない
- 有効性を確認した正式URLだけを掲載している
- 旧コードを流用していない
- テンプレート感が強くなく、ROOMのコンセプトと素材に根拠がある

### 対応環境とレイアウト

- GitHub Pagesの`/ryu-ikon-room/`サブパスで全ページ・全アセットが正常動作する
- PC、スマートフォン、タブレットで横はみ出し、重なり、欠落がない
- 縦向き・横向き、ズーム、長い日本語表示で主要情報が破綻しない
- JavaScriptや外部フォントの読み込み失敗時も、確定情報と主要導線を利用できる

### アクセシビリティ

- `prefers-reduced-motion`で不要な動きを停止または置換できる
- キーボードだけで全リンク・操作要素を利用でき、フォーカスが見える
- 文字、ボタン、背景のコントラストが適切である
- 見出し階層、ランドマーク、ページタイトル、lang、リンク名が適切である
- 内容画像に目的に合うalt、装飾画像に空altがある
- 動画には自動再生の制御、音声制御、poster/代替説明など必要なフォールバックがある

### パフォーマンスと素材

- 画像の表示サイズ、形式、圧縮、width/height、遅延読み込みを用途ごとに最適化する
- HeroなどLCP候補の読み込み優先度が適切である
- 動画はモバイル回線と低性能端末を考慮し、静止画フォールバックを持つ
- 不要な継続アニメーションやメインスレッド負荷を残さない
- 使用素材の権利、クレジット、被写体の許諾を確認する

### SEO・共有・公開

- OGP（title、description、image、url、type）が正しい
- faviconが主要ブラウザで表示される
- canonicalが本番URLを指す
- `robots.txt`が公開方針と一致する
- `sitemap.xml`が公開ページと一致する
- 専用404がサブパスで正常表示・移動できる
- MusicEvent構造化データが確定情報だけを含み、検証に通る
- 内部リンクと外部リンクを公開直前に確認する

### 品質・復元性

- 採用した技術のbuildが成功する
- 採用した技術のlintが成功する
- HTML/アクセシビリティ/リンクに重大エラーがない
- `main`切替前に`room-v1-archive`から旧公開状態を復元できることを確認する
- 公開切替は人間が差分、プレビュー、rollback手順を確認して承認する

## 数値目標の候補（現時点では絶対条件ではない）

- LighthouseのPerformance / Accessibility / Best Practices / SEOを各90以上にする候補
- Core Web Vitalsの「良好」範囲を目標にする候補
- モバイル初期転送量や動画容量の上限を素材決定後に設定する候補
- 主要幅の候補: 360px、390px、768px、1024px、1440px
- キーボード、200%ズーム、reduced motion、低速回線を公開前確認セットに含める候補

数値は技術選定、素材、計測条件をClaudeと人間が確認した後に採否を決めます。
